# 普通脚本编程使用指南

这份说明只适用于普通脚本，不适用于植物大战僵尸2专用脚本。普通脚本没有植物卡槽、棋盘、阳光、能量豆等校准变量。

本文件给两类对象使用：

- 用户：看完后知道怎么写普通脚本。
- AI：看完后知道怎么帮助用户写普通脚本，并避免把 PVZ2 专用内容混进来。

## 1. 基本概念

普通脚本使用 Lua 语法，按顺序执行代码。

脚本里的坐标默认使用屏幕百分比坐标，范围是 `0` 到 `100`，可以写小数。

示例：

```lua
tap(50, 50)
```

含义是在屏幕正中央点击。

`x = 0` 表示屏幕最左边，`x = 100` 表示屏幕最右边。  
`y = 0` 表示屏幕最上边，`y = 100` 表示屏幕最下边。

常用写法：

```lua
tap(15.35, 46.94)
wait(500)
swipe(20, 80, 80, 80, 300)
```

### 运行前权限准备

首次安装或重新安装后，需要在主界面完成以下授权：

- 无障碍服务：用于执行点击和滑动。
- 悬浮窗权限：用于显示录制、编辑和运行控制面板。
- 屏幕捕获：用于颜色检测和 OCR 文字检测。

授权屏幕捕获时，如果系统支持“仅共享一个应用”，推荐只选择目标游戏。这样 OCR
只会截取游戏画面，不会把本应用悬浮窗识别进去。屏幕捕获被关闭或系统回收后，需要
回到主界面重新授权。

## 2. 点击

点击使用 `tap`。

```lua
tap(x, y)
tap(x, y, 持续时间)
```

参数说明：

- `x`：点击位置的横向百分比。
- `y`：点击位置的纵向百分比。
- `持续时间`：按下持续时间，单位毫秒，可省略。

示例：

```lua
tap(50, 50)
tap(50, 50, 30)
```

`tap(50, 50, 30)` 表示在屏幕中心按下约 30 毫秒。

建议：

- 普通按钮点击一般用 `tap(x, y)` 或 `tap(x, y, 1)`。
- 如果按钮响应不稳定，可以用 `tap(x, y, 30)`。
- 第三个参数不是点击次数，不要写成 `tap(x, y, 5)` 来表示点 5 次。

多次点击应这样写：

```lua
tap(50, 50)
wait(100)
tap(50, 50)
wait(100)
tap(50, 50)
```

## 3. 滑动

滑动使用 `swipe`。

```lua
swipe(起点x, 起点y, 终点x, 终点y, 持续时间)
```

参数说明：

- `起点x`、`起点y`：滑动开始位置，百分比坐标。
- `终点x`、`终点y`：滑动结束位置，百分比坐标。
- `持续时间`：滑动持续时间，单位毫秒。

示例：

```lua
swipe(20, 80, 80, 80, 300)
```

建议：

- 普通滑动推荐 `100` 到 `500` 毫秒。
- 太快可能无效，太慢会影响脚本速度。
- 如果省略第五个参数，程序会默认使用 `300` 毫秒，但建议明确写出来。

## 4. 等待

等待使用 `wait`。

```lua
wait(毫秒)
```

示例：

```lua
wait(500)
tap(50, 50)
```

建议：

- 页面切换、动画、加载后都要加等待。
- 循环里必须有 `wait(...)`，否则脚本可能占用过高并卡住。
- 循环里最小建议 `wait(10)`，普通检测建议 `wait(50)` 到 `wait(500)`。

## 5. 文字检测

新脚本统一使用 `check_text` 和 `check_text_not`。

```lua
check_text("文字", 左, 上, 右, 下)
check_text_not("文字", 左, 上, 右, 下)
```

参数说明：

- `"文字"`：要查找的文字。
- `左, 上, 右, 下`：检测区域，全部是百分比坐标。

示例：

```lua
if check_text("开始", 0, 0, 100, 100) then
    tap(50, 80)
end
```

表示如果全屏范围内检测到“开始”，就点击屏幕下方。

限定区域示例：

```lua
if check_text("继续", 30, 70, 70, 95) then
    tap(50, 85)
end
```

表示只在屏幕下半部分的一块区域找“继续”。

`check_text_not` 用于文字没有出现时执行：

```lua
if check_text_not("加载中", 0, 0, 100, 100) then
    tap(50, 50)
end
```

注意：

- 新脚本不要主动使用 `ocr_text` 或 `ocr_text_not`。
- `ocr_text`、`ocr_text_not` 只作为旧脚本兼容方法存在。
- AI 帮用户写普通脚本时，文字检测只推荐 `check_text` / `check_text_not`。
- 当前版本使用离线 PaddleOCR，并且始终识别原始游戏截图，没有灰度、黑白或反色滤镜。
- 每次调用都会截取当时的游戏画面，因此可以检测动态变化的文字。
- 第一次文字检测可能需要初始化 OCR；后续检测通常更快。
- OCR 比普通点击和颜色检测耗时，循环文字检测建议使用 `wait(200)` 到 `wait(500)`。

## 6. 颜色检测

颜色检测使用 `check_color` 和 `check_color_not`。

```lua
check_color("#颜色", 容差, x, y)
check_color_not("#颜色", 容差, x, y)
```

参数说明：

- `#颜色`：十六进制颜色，例如 `#FF0000`。
- `容差`：允许颜色误差，常用 `10`。
- `x, y`：检测点位置，百分比坐标。

示例：

```lua
if check_color("#FF0000", 10, 50, 50) then
    tap(50, 50)
end
```

表示如果屏幕中心点颜色接近红色，就点击屏幕中心。

颜色不匹配时执行：

```lua
if check_color_not("#000000", 10, 50, 50) then
    tap(50, 50)
end
```

建议：

- 颜色可以用编辑器里的颜色识别工具选取。
- 容差一般从 `10` 开始，不稳定时可以调到 `20` 或 `30`。
- 检测点要选在颜色稳定的位置，不要选渐变、动画、阴影边缘。

## 7. 条件判断

Lua 中使用 `if ... then ... end`。

文字出现后点击：

```lua
if check_text("开始", 0, 0, 100, 100) then
    tap(50, 80)
end
```

颜色匹配后点击：

```lua
if check_color("#00FF00", 10, 60, 40) then
    tap(60, 40)
end
```

多个条件可以用 `and`：

```lua
if check_text("开始", 0, 0, 100, 100) and check_color("#00FF00", 10, 50, 50) then
    tap(50, 50)
end
```

多个条件可以用 `or`：

```lua
if check_text("开始", 0, 0, 100, 100) or check_text("继续", 0, 0, 100, 100) then
    tap(50, 80)
end
```

## 8. 循环

循环用于反复检测和执行。

无限循环：

```lua
while true do
    tap(50, 50)
    wait(500)
end
```

循环检测文字：

```lua
while true do
    if check_text("开始", 0, 0, 100, 100) then
        tap(50, 80)
    end
    wait(200)
end
```

固定次数循环：

```lua
for i = 1, 5 do
    tap(50, 50)
    wait(300)
end
```

强制要求：

- 任何 `while true do` 循环里都必须有 `wait(...)`。
- 任何长期运行的循环里都必须有 `wait(...)`。
- AI 写脚本时，不允许生成没有 `wait(...)` 的死循环。

## 9. 并行执行两个动作

如果需要同时执行两路动作，使用 `parallel`。

```lua
parallel(
    function()
        tap(25, 50)
    end,
    function()
        tap(75, 50)
    end
)
```

两路都可以包含多个动作：

```lua
parallel(
    function()
        tap(25, 50)
        wait(200)
        tap(25, 60)
    end,
    function()
        swipe(80, 80, 80, 20, 500)
    end
)
```

常见错误：

```lua
parallel(function()
    tap(25, 50)
end function()
    tap(75, 50)
end)
```

上面是错误写法，因为两个 `function() ... end` 中间缺少逗号。

正确写法：

```lua
parallel(
    function()
        tap(25, 50)
    end,
    function()
        tap(75, 50)
    end
)
```

规则：

- 每一路必须写成 `function() ... end`。
- 多路之间必须用英文逗号 `,` 分隔。
- 最后一个 `end` 后面不需要逗号。
- 并行里也可以写 `wait(...)`、`if`、`while`。

## 10. 完整示例

等待按钮出现后点击：

```lua
while true do
    if check_text("开始", 0, 0, 100, 100) then
        tap(50, 80)
        break
    end
    wait(200)
end
```

点击后等待页面变化：

```lua
tap(50, 80)
wait(1000)

if check_text("确认", 30, 60, 70, 90) then
    tap(50, 75)
end
```

颜色变成指定颜色后点击：

```lua
while true do
    if check_color("#00FF00", 10, 50, 50) then
        tap(50, 50)
        break
    end
    wait(100)
end
```

同时执行两个动作：

```lua
parallel(
    function()
        tap(30, 50)
        wait(100)
        tap(30, 60)
    end,
    function()
        swipe(80, 80, 80, 20, 300)
    end
)
```

## 11. 常见问题

### 点击位置不准

检查坐标是否是百分比坐标，而不是像素坐标。

正确：

```lua
tap(50, 50)
```

不推荐：

```lua
tap(720, 1600)
```

普通脚本里 `tap(720, 1600)` 会被当成百分比，超出范围后结果不可控。

### 循环卡住

检查循环里是否有 `wait(...)`。

错误：

```lua
while true do
    if check_text("开始", 0, 0, 100, 100) then
        tap(50, 80)
    end
end
```

正确：

```lua
while true do
    if check_text("开始", 0, 0, 100, 100) then
        tap(50, 80)
    end
    wait(200)
end
```

### 文字检测不到

优先检查：

- 是否已经在主界面授权屏幕捕获；支持时应只选择目标游戏。
- 识别区域是否框对。
- 文字是否完整写对。
- 是否需要缩小识别区域，避免干扰。
- 文字是否正在动画、模糊或被其他游戏元素遮挡。

文字条件与编程编辑器中的文字框选工具说明：

- 动作条件里的文字识别：框选范围后会尝试识别当前画面，并自动填写匹配文字。
- 编程编辑器里的文字识别工具：只负责插入
  `check_text("文字", 左, 上, 右, 下)`，需要手动把 `"文字"` 改成目标内容。

### 颜色检测不稳定

优先检查：

- 检测点是否选在动画或渐变区域。
- 容差是否太小。
- 颜色是否会因亮度、特效、状态变化而改变。

## 12. AI 助手使用规则

AI 帮用户写普通脚本时必须遵守：

- 只使用普通脚本能力，不要使用 PVZ2 专用校准变量。
- 不要写 `plant_slots`、`board`、`sun_`、`plant_food_`、`artifact_`、`cucumber_`、`recharge_`、`cards_edge`、`other_` 等 PVZ2 变量。
- 坐标一律按百分比 `0` 到 `100` 编写。
- 点击使用 `tap(x, y)` 或 `tap(x, y, durationMs)`。
- 滑动使用 `swipe(sx, sy, ex, ey, durationMs)`，建议写第五个参数。
- 文字检测只推荐 `check_text` 和 `check_text_not`。
- 不主动使用 `ocr_text` 和 `ocr_text_not`，除非用户明确要求兼容旧脚本。
- 颜色检测使用 `check_color` 和 `check_color_not`。
- 所有长循环都必须包含 `wait(...)`。
- 如果用户要同时做两个动作，使用 `parallel(function() ... end, function() ... end)`。
- 如果用户描述的是屏幕位置，先帮用户转成百分比坐标，或建议使用编辑器里的点击/滑动/选区/颜色工具插入。
- 不要把注释头、脚本名、循环次数等导出元信息写进用户脚本正文。

## 13. AI 可直接套用的最小模板

```lua
wait(1000)

while true do
    if check_text("开始", 0, 0, 100, 100) then
        tap(50, 80)
        break
    end
    wait(200)
end
```
